# Problem

給你一個 `mxn` 的矩陣，問你能不能找到 `target` 在哪一個行哪一列?
其中每一列都是由小到大排序的，第 $i+1$ 列的第一個大於第 $i$ 列的最後一個數字

實作複雜度限制在: $O(\log{nm})$

## 測資限制

- $1 \le n, m \le 100$
- $-10^4 \le mat[i][j] \le 10^4$

# 想法

因為是由小到大排序的，且題目限制在$O(\log{nm})$ 所以直覺想到二分搜
先對第一行二分搜，找到 index $i$，再對第 $i$ 列做二分搜，找到 index $j$

- 時間複雜度: $\mathcal{O}(\log{nm})$
- 空間複雜度: $\mathcal{O}(1)$
